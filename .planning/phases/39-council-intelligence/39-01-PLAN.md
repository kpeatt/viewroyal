    1. Query the `topics` table to get topic name -> ID mapping
    2. Query all agenda items with their categories: `SELECT id, category FROM agenda_items`
    3. For SQL-mapped items: Use RPC call to `normalize_category_to_topic(category)` or replicate the mapping in Python by querying the function for all distinct categories first, then batch-inserting into `agenda_item_topics`. Efficient approach: run a single SQL INSERT...SELECT using the function:
       ```sql
       INSERT INTO agenda_item_topics (agenda_item_id, topic_id)
       SELECT ai.id, t.id
       FROM agenda_items ai
       JOIN topics t ON t.name = normalize_category_to_topic(ai.category)
       WHERE NOT EXISTS (SELECT 1 FROM agenda_item_topics ait WHERE ait.agenda_item_id = ai.id)
       ```
       Execute via `supabase.rpc()` or raw SQL.
    4. For unmapped items (those still without a topic after step 3): collect their distinct categories, batch-classify with a single Gemini call (all ~170 unmapped categories fit in one prompt asking Gemini to map each to one of the 8 topics), then bulk-insert results.
