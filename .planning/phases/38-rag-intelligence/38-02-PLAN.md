<objective>
Add LLM-based reranking of tool results before they reach the agent context, with trace logging and UI visibility.

Purpose: Raw vector/keyword search results contain noise. Reranking with Gemini Flash Lite scores each result's relevance to the user's question and filters out low-quality evidence, giving the agent cleaner context for better answers. The reranking step is logged to traces for observability and shown in the research steps UI.

Output: `rerankResults` function in rag.server.ts, reranking wired into agent loop, new "reranking" AgentEvent type, trace metadata, and UI display.
</objective>

**2. Create `summarizeResult` helper:**

- Motions: `[Motion] ${result.plain_english_summary?.slice(0, 120) || result.motion_text?.slice(0, 120)}`

- Default: `JSON.stringify(result).slice(0, 120)`

**3. Create `rerankResults` function:**
```typescript
interface RerankScore { index: number; score: number; }
interface RerankOutput { kept: number[]; dropped: number[]; scores: RerankScore[]; latency_ms: number; }

async function rerankResults(
  query: string,
  results: any[],
  threshold: number = 3,
  minKeep: number = 3,
): Promise<RerankOutput>
```

Implementation:
- If results.length <= minKeep or getGenAI() is null, return all indices as kept with score 10, latency 0.
- Build prompt: "Rate each search result's relevance to the question on a scale of 0-10. Question: '${query}'. Results: [0] summary... [1] summary... Return JSON: {\"scores\": [{\"index\": 0, \"score\": 8}, ...]}"
- Use `summarizeResult` to create ~100-150 char summaries per result.
- Call `getGenAI()!.models.generateContent({ model: "gemini-2.5-flash-lite", contents: [prompt], config: { responseMimeType: "application/json" } })`.
- Parse JSON response. Sort by score descending.
- Keep results above threshold OR until minKeep is reached (always keep at least minKeep).
- Track latency (Date.now() before/after the API call).
- Wrap in try/catch -- on failure, return all results as kept (graceful degradation).

**4. Create `flattenToolResults` helper:**
Takes a consolidated tool result (object with named arrays) and returns a flat array with type labels preserved. For search_council_records: flatten motions + transcripts + statements + agenda_items into a single array, each item getting a `_result_type` property. **5. Create `unflattenRerankResults` helper:**

let rerankingMeta: { candidates: number; selected: number; model: string; latency_ms: number; threshold: number } | undefined;

  const flat = flattenToolResults(tool_name, toolResult);
  if (flat.length > 3) {  // Only rerank if enough results to filter
