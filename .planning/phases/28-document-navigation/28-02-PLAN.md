**1. Create `apps/web/app/lib/cross-references.ts`:**

This is a pure utility module (no React, no Supabase) that detects bylaw references in text and resolves them against a known list.

```typescript
export interface CrossReference {
  pattern: string;         // The matched text, e.g., "Bylaw No. 1059"
  bylawNumber: string;     // The extracted number, e.g., "1059"
  targetType: "bylaw";     // Future: could expand to "document"
  targetId: number;        // bylaws.id
  targetTitle: string;     // bylaws.title
  targetUrl: string;       // "/bylaws/{id}"
  sectionOrders: number[]; // Which section_orders contain this reference
}

interface BylawRecord {
  id: number;
  title: string;
  bylaw_number: string;
}
```

Export a function `detectCrossReferences`:
```typescript
export function detectCrossReferences(
  sections: Array<{ section_text: string; section_order: number }>,
  bylaws: BylawRecord[],
): CrossReference[]
```

Implementation:
1. Build a `Map<string, BylawRecord>` from `bylaws` array, keyed by `bylaw_number`
2. Define regex: `const BYLAW_RE = /\bBylaw\s+No\.\s*(\d+)(?:\s*,\s*\d{4})?\b/gi;`
3. For each section, run the regex against `section_text`
4. For each match, extract the number and look it up in the map
5. Only create a CrossReference if the bylaw exists in the database (filters false positives)
6. Deduplicate: if the same bylaw is referenced in multiple sections, merge into one CrossReference with multiple `sectionOrders`
7. Sort output by first occurrence (lowest sectionOrder first)
8. Build `targetUrl` as `/bylaws/${bylaw.id}`

Edge cases:
- "Bylaw No. 1059, 2020" -- extract just "1059", ignore the year suffix
- "Bylaw No. 35" -- short numbers work fine
- "Bylaw No. 9999" -- not in database, filtered out
- Same bylaw referenced 5 times across sections -- one CrossReference entry with all sectionOrders
