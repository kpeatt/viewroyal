**1. Create `apps/web/app/lib/use-scroll-spy.ts`:**

A custom React hook that uses IntersectionObserver to track which section is currently visible at the top of the viewport.

```typescript
// useScrollSpy(sectionIds: string[]): string | null
```

Implementation details:
- Accept an array of section element IDs (e.g., `["section-1", "section-2", ...]`)
- Create an IntersectionObserver in a `useEffect` with `rootMargin: "0px 0px -80% 0px"` and `threshold: 0` -- this makes only the top 20% of the viewport count as the "active zone"
- When multiple sections are intersecting, pick the one whose `boundingClientRect.top` is closest to the top of the viewport (smallest positive value)
- Return the active section ID as state (`string | null`)
- Initialize as `null` -- NO server-side rendering of active state (avoids hydration mismatch)
- Clean up observer on unmount
- Re-create observer if `sectionIds` array changes (use JSON.stringify for stable deps)

**scrollToSection helper function (inside component):**
```typescript
function scrollToSection(sectionId: string) {
  const el = document.getElementById(sectionId);
  if (!el) return;
  el.scrollIntoView({ behavior: "smooth", block: "start" });
  history.replaceState(null, "", `#${sectionId}`);
}
```

IMPORTANT: The DocumentTOC component will need to detect its context (mobile vs desktop) via props or by rendering both views internally. The simpler approach is to have DocumentTOC always render BOTH the desktop nav and mobile bar internally, and let the parent use `className="lg:hidden"` / `className="hidden lg:block"` wrappers. However, since the desktop sidebar needs the `sticky` wrapper in the parent, the cleanest approach is:

- The DocumentTOC component renders the TOC list (used in both contexts)
- The mobile bar wrapper (with expand/collapse) is part of DocumentTOC when used in the mobile slot
- For desktop, DocumentTOC renders just the nav list

Alternative cleaner approach: Have DocumentTOC accept a `variant` prop: `"desktop" | "mobile"`.
- `variant="desktop"`: Renders the nav list inside a ScrollArea (no sticky -- parent handles that)
- `variant="mobile"`: Renders the sticky top bar with collapsible dropdown

This avoids rendering duplicate DOM. Use `variant="mobile"` in the `lg:hidden` wrapper and `variant="desktop"` in the `hidden lg:block` wrapper.
